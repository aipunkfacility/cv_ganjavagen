# Результаты отладки скролла (/debug-astro)

Процесс отладки был выполнен на основе специализированного навыка `debug-astro-site`.

## Выполненные шаги

1. **Диагностика конфликтов**:
   - Обнаружен конфликт между CSS-свойством `scroll-behavior: smooth` и вызовом JS `scrollTo({ behavior: 'smooth' })`.
   - Выявлено, что принудительный возврат к `scroll-snap-type: mandatory` происходил слишком рано (по таймеру), что вызывало «прыжок» при возврате наверх.

2. **Внесенные изменения**:
   - **CSS**: Удалено свойство `scroll-behavior: smooth` из файла [index.css](file:///d:/Dev/Projects/CV/src/styles/index.css) для основного контейнера, чтобы не мешать программному управлению.
   - **JS**: В файле [Layout.astro](file:///d:/Dev/Projects/CV/src/layouts/Layout.astro) реализована новая логика обратной связи. Теперь "магнит" (Snap) включается обратно только тогда, когда контейнер физически достиг отметки `0`.
   - **Очистка**: Удален временный кэш `.astro` для чистого рестарта.

## Результат

Кнопка «Наверх» теперь должна работать идеально плавно, не конфликтуя с прилипанием блоков в других частях страницы.
